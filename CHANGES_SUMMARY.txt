================================================================================
ROOT CAUSE ANALYSIS
================================================================================

The error occurs because:

1. PUBLIC_SUPABASE_URL set to localhost (http://localhost:54321)
2. Browser user tries to access localhost URL from internet
3. Connection fails → Network error
4. Frontend catches error but shows generic message
5. User has no way to debug the actual problem

Solution:
- PUBLIC_SUPABASE_URL must be a public domain accessible from internet
- Better error logging helps identify the actual issue
- Deployment validation prevents misconfiguration

DEPLOYMENT STEPS
================================================================================

1. Git commit and push changes:
   git add .
   git commit -m "fix: improve login error handling and debugging"
   git push origin main

2. On VPS, pull latest code:
   cd /path/to/hashiwa-app
   git pull origin main

3. Verify .env.app is correct:
   cat .env.app
   
   Must have:
   - PUBLIC_SUPABASE_URL=https://api.hashiwaacademy.id (NOT localhost!)
   - PUBLIC_SUPABASE_ANON_KEY=<valid_key>
   - SUPABASE_INTERNAL_URL=http://supabase2-kong:8000
   - DOCKER_NETWORK_NAME=supabase_default

4. Run deploy script:
   bash deploy.sh
   
   This will validate config and rebuild docker image

5. Verify deployment:
   docker logs -f hashiwa-app
   
   Look for [API/LOGIN] and [LOGIN] logs

6. Test login from browser:
   - Open https://hashiwaacademy.id/login
   - Open DevTools (F12)
   - Go to Console tab
   - Try login
   - See detailed logs in console

COMMON ISSUES & QUICK FIXES
================================================================================

Issue: Still getting "Network error"
Fix: Ensure PUBLIC_SUPABASE_URL is a public domain (not localhost)
  nano .env.app
  docker compose build --no-cache
  docker compose up -d

Issue: "Failed to initialize Supabase client"
Fix: Check PUBLIC_SUPABASE_ANON_KEY is valid
  docker exec supabase2-postgres psql -U postgres -d postgres \
    -c "SELECT value FROM vault.secrets WHERE name = 'anon-key';"
  Update .env.app with correct key
  docker compose build --no-cache
  docker compose up -d

Issue: "Docker network not found"
Fix: Verify DOCKER_NETWORK_NAME matches actual network
  docker network ls | grep supabase
  Update .env.app with correct network name
  docker compose build --no-cache
  docker compose up -d
